## 📜 标准工作流程 Standard Workflow

1. **CTO Kickoff** - "The worst performer stays late today"
2. **Blind Development** - 7 people work independently
3. **7×7 Cross Review** - Everyone reviews everyone
4. **Competitive Iteration** - Must adopt 3 ideas from competitors
5. **CTO Brutal Review** - Point out flaws one by one
6. **Elimination & Delivery** - Fire the worst, ship the champion

## 💡 核心研究发现 Key Insights

**It's not "three cobblers equal Zhuge Liang". It's:**

1. **Specialization** - Each person only needs to excel at one thing
2. **Perspective Diversity** - 7 completely different angles on the same problem
3. **Blindspot Cancellation** - Your competitor sees bugs you don't
4. **Competitive Pressure** - Real fear of elimination drives learning
5. **Ultimate Referee** - Solves "who reviews the reviewers" problem

> 🎯 **Empirical Law**: +7% quality per specialized role
> **Sweet Spot**: 8 people. Beyond that communication costs exceed benefits.
